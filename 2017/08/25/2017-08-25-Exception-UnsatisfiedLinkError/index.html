解决方案

对于Android Studio

  • 在工程目录下gradle.properties(Global Properties)里面添加android.useDeprecatedNdk=true

变量配置包括：全局配置、Project配置、Module配置

方案分析

useDeprecatedNdk

useDeprecatedNdk 属性的添加不是必须的，从命名可以看出，这是一个兼容性配置 允许兼容低版本(或废弃掉)的NDK
本地NDK较新的话，注释掉这句代码，程序也可以正常编译的

abiFilters

参考了"CSDN@只是备忘"的博客：http://blog.csdn.net/wove55678/article/details/52313208

abiFilters也是一个兼容配置
在引入第三方项目时，如果第三方项目兼容了“x86”，而本地项目目录中没有“x86”目录，便会抛出UnsatisfiedLinkError
//TODO 为何此时系统会只在"x86"中查找而不遍历其他目录 从错误名上可以看出[satisfy：满足]，是没有兼容的错误，但是程序运行时却不通过。

上面配置的意思是指定ndk需要兼容的架构，把除了armeabi和v7a以外的兼容包都过滤掉，只剩下armeabi和v7a的文件夹。

  • armeabi 性能较差，但是兼容性较好
  • v7a 对于浮点计算的cpu来说性能更好，不兼容不支持浮点的cpu

很多普及的APP只配置了armeabi 无论如何，abiFilters还是应该添加的

JNI的java接口类所在的包名必须与JNI层的命名规则一致
