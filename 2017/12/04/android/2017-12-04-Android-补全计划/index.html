Android 补全计划

发表于 2017-12-04 | 阅读次数

这几年android进阶缓慢，弊端呈现，心有所欲。
于是，参考了知乎的问答
Android开发初期之后怎么提升？怎么才能叫精通？方向在哪？
取几个我有所收益的答主，作为记录。

知乎答主贴

hi大头鬼hi

“并非越底层越牛逼”，想想你的过往，那些偏科的学生当真赢得了学涯巅峰？
点与面的关系：不要为了一颗树放弃整个森林，要经常跳出来，飞出来，看看这久往的星空，咦，不经意间看一下原来所在的森林哈。
如此，明确正确的钻研方向：整体是一方面，android真正领域在哪里？解决问题(业务)、用户交互、性能。
在技术上方向大致为：源码 -> android系统机制 -> 横向反哺(其他语言，其他技术，后台前端等)

李板溪

android知识点列举
作者：李板溪
链接：https://www.zhihu.com/question/28425576/answer/40862900
来源：知乎
著作权归作者所有。商业转载请联系作者获得授权，非商业转载请注明出处。

一: 开发技能
你可以看成是一般的编程技能这方面你可以从
编程语言的熟练掌握
面向过程编程思想
面向对象的思想
函数式编程思想
设计模式
算法与数据结构
网络编程,TCP/IP 协议
重构
版本管理(Git 等)
等方面的检查和提高自己
更详细的技能树,请自己搜索
二: 移动应用开发
你可以看成是 App 开发 估计这可能是你更感兴趣的 这方面你可以从:
MVC 这成三个方面来对照检查下
M: 数据层
移动应用数据结构特点
数据存储 :SQLite数据库,文件存储
数据格式: XML 格式,JSON格式
序列化与反序列化
数据查询: 移动应用一般数据库主要是 用SQLite (说回来,单是 SQLite 数据库,就可以花不少时间来深入学习下, 因为对一个应用来说,数据基本是核心)
V: 视图层
移动应用界面特点
移动应用构建界面常用方法
移动应用交互特点
移动应用动画
系统 UI 系统特点,优点,缺点及局限
C: 控制层
移动应用控制层特点
控制层的生命周期
多线程,UI 线程,后台线程
再加一层:E: 事件层
事件处理,触控事件,手势,事件响应链
三: Android 开发 工具篇 - 工欲善其事,必先利其器
Android Studio 掌握用得怎么样了?
Adb 及相关工具会用吗?
Gradle 构建系统呢?
文档篇 - 看 Android 官方是怎么定义开发各种技能的.
相信常看 Android 开发者官方网站,你会收益良多,我下面写的也不必看了.
系统篇
Android 多线程编程,异步编程特点 - Loop,Handler,IntentService,Broadcast,MessageQueue
Android UI 框架特点,优势和不足
然后你再按 MVC 将 Android 各部分再分析分析,总结总结.

Tinker

Google Sample提供了全面的android知识点
跟着下面的步骤来：

  1. Google Sample里面的代码全跑一遍
  2. 读Google Sample里面的源码
  3. 背下所有的继承关系

关于第三点可以说是简单粗暴，不过，以前看设计模式也是为了更好的看代码结构。
这种东西就像是酒醇，可预见的收益。
关于精通的另类解释：
能回答小白的所有android问题
另外还提到了android bug的处理方式：

  1. 流程检查(逻辑)
  2. 控件bug -> 读源码
  3. 通过继承解决(源码的基础上做修正)
  4. 使用google + site + keywords

全照搬不如不看，这类整理是非常好的参考，但是要基于自己最后写出一套自己的。

总结

android主体还是android的知识点、android的知识体系。
至于是先体系还是先tools，我认为是相辅相成的，但主次还是很明确的，知识体系为主，tools为辅。
那么确定下的
语数外：android知识体系
关键词：Google Sample，继承关系
理化生：
android tools，adb tools， Android Studio tools,
数据结构，算法，设计模式
体美劳：
其他语言，MAC技术，协议，网络
进阶：
系统机制，性能，精通模块，精通Android

对商人来说，察觉对方想要什么是必须具备的技能
