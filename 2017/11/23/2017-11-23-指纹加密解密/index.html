初始化密钥生成器

先上代码

初始化密钥生成器，需要指定一个算法标准，依据不同的版本： // KEY_ALGORITHM_RSA (18+)
// KEY_ALGORITHM_AES (23+)
RSA是非对称加密，生成公钥和私钥，分为签名密钥对和加密秘钥对
签名：私钥签名，公钥验签
加密：公钥加密，私钥解密
AES是对称加密，执行速度快，易于硬件实现。由于是对称加密，密钥需要在传输前通讯双方获知。
PS： 基于以上特点，通常使用RSA来首先传输AES的密钥给对方，然后再使用AES来进行加密通讯。 初始化密钥生成器的第二个参数是provider，指定为之前我们完成实例化的KeyStore类型即可。

配置密钥生成器

直接上Google代码

创建一对密钥时，需要给密钥对一个别名：keyName
密钥的使用场景：PURPOSE_ENCRYPT(加密) or PURPOSE_DECRYPT(解密)
加密模式：BLOCK_MODE_CBC(使用CBC)加密
什么情况下被授权使用密钥对：UserAuthenticationRequired(用户认证通过后，即指纹)
数据位数不够时的填充模式：ENCRYPTION_PADDING_PKCS7(这个我也不是很明确怎么填充的)
用户指纹变更了密钥对是否还有效：InvalidatedByBiometricEnrollment
配置完毕后，执行密钥生成器的初始化
