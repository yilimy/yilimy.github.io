注意事项

  1. 执行连接，是一个耗时的操作，需要在非主线中执行
    主线并没有强制禁止耗时任务，即理论上可执行连接，但在主线程的阻塞会引发一些意外导致connect失败
  2. 执行连接操作时，需要停止设备搜索
    blueAdapter.cancelDiscovery()
    详情转BluetoothSocket.connect()的API文档
  3. 获取远程设备代理时，需要一个UUID，按照createInsecureRfcommSocketToServiceRecord接口描述
    标准的蓝牙SPP_UUID:00001101-0000-1000-8000-00805F9B34FB
    两个设备只有在UUID一致时，才能进行连接
    部分连接失败或连接超时[1]，可能是由UUID不一致导致的
  4. 设备支持的UUID是不确定的，可以在连接之前，进行设备配对
    devices[i].createBond()
    这样就能得到设备所支持的UUID列表了
    var _uuid = devices[i].uuids
    我发现蓝牙耳机不支持标准蓝牙UUID，什么情况? // TODO
    它支持的UUID之一：音频 "00001108-0000-1000-8000-00805F9B34FB"
    更多的UUID参考：蓝牙服务对应的UUID码
  5. 连接成功了，状态栏并没有像系统的蓝牙连接那样，出现图标变动，什么情况？ // TODO
