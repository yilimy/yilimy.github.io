简介

国内的评论需要实名，需要备案，emmm…好在imsun实现的一款基于 GitHub Issues 的评论系统Gitment，很是厉害的。之后，yanzi1225627又对其做了改进，使其不用在针对性添加评论标签(作者文档上说在需要的页面上添加)，而是统一配置于每一页面。评论支持md，比较适合程序员。
Gitment 是作者实现的一款基于 GitHub Issues 的评论系统。支持在前端直接引入，不需要任何后端代码。可以在页面进行登录、查看、评论、点赞等操作，同时有完整的 Markdown / GFM 和代码高亮支持。尤为适合各种基于 GitHub Pages 的静态博客或项目页面。
Gitment 只能使用 GitHub 账号进行评论。

备注

同一篇文章需要初始化comment两次的问题，是因为http://xxx.com/post/ab9bb85a.html和点击阅读全文进去的链接http://xxx.com/post/ab9bb85a.html#more对issues来说是不同的，所以创建两次。
解决方法就是gitment.swig里id弄成window.location.pathname而不是document.location.href。

gitment可能不支持链接地址里有中文，所以安装gitment前一定要把链接持久化搞成全是英文的。
http://yangq.me/post/19acb239.html

window.location.pathname依旧是有问题的，github在生成issue的时候对长度有限制
可替换成’<%= page.title %>'
